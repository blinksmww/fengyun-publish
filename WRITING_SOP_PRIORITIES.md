## 🔴 P0 铁律(违反直接扣 ≥10 分 / 进扑街区)

### 1. 标题必带英文产品名(深挖 6 / 9 跨账号 4/4 一致)

- ✅ Claude / Anthropic / Skills / Claude Code / Vibe Coding(2026 唯一通过 v2 验证的真规律)
- ❌ 严禁标题里写 OpenAI / GPT / Veo / GLM(标题命中爆款率 -9% ~ -47%)
- ✅ 标题英文字数 ≥ 8 字符(扑街红线之一,英文<7 字危险)

### 2. 一行一段比例严格 <30%(深挖 5 + 8 致命组合)

- 一行一段 <10% → 爆款率 **53.4%**
- 一行一段 30-50% → 爆款率 47.3%
- 一行一段 >70% → 爆款率 **18.1%(死亡区)**
- ⚠️ 致命组合:**短英文标题 + 一行一段>50% + 短词** → 扑街率 **4.5× 基线**

### 3. 文章字数 3500-5500 字 sweet spot

- 5500-8000 字也行但 ceiling
- <500 字 / >20000 字 扑街
- 2026 爆款中位 = **4078 字**(不是 4100,这是个 range 不是 point)
- ⚠️ **手机端 4000 字以下读起来更舒服**(待 Phase 3-1 调研验证)

### 4. 段落写长 + 错落(跨期稳定 Top 5)

- 平均段长 **≥200 字**(每千字 <5 段)
- 段长 std 大(错落感强 = 节奏感)
- 首段 50+ 字(完整段落,不要一句话开场)
- 末段也要写长(viral_ending_strength 真规律 mean_ρ=+0.235)

### 5. 封面必须有 + 色调(深挖 7 跨账号 4/4 一致)

- ✅ 暖橙(15-45° 色相)/ 白底 / 深色海报
- ❌ 蓝色(4/4 跨号扣分)
- ❌ 红色(4/4 跨号扣分)
- ❌ 极饱和(>0.70)/ 绿 / 青 / 粉
- ⚠️ 必须 has_cover_color = 1(强效应 d=0.69)

## 🆕 Phase 3-1 + 3-2 调研新增规则(2026-05-21 update)

### 手机端阅读硬数据(Phase 3-1)

```
✅ 字号:       15 px(微信公众号主流)/ 学术最优 16-19px
✅ 行高:       1.75
✅ 单段最长:    80-150 字 / 7 行上限 ⚠️ 跟 SOP "200+ 字" 冲突
✅ 每屏字数:    300-500 字
✅ 配图频率:    每 1000-1200 字一张(不是 SOP 的 15+ 张)
✅ 头部 200 字: 前 2 句必须钩子,100 字内明确价值承诺
✅ 完读率:      每多 1000 字 -10%
✅ 学术甜点:    1500-2000 字(参与度 1.55× 基线)
```

### 数据 SOP vs 学术 SOP 的取舍

| 维度 | SOP v2.1 数据派 | 学术研究 | 折中建议 |
|---|---|---|---|
| 字数 | 3500-5500 | 1500-2000 | **2500-4000 字** |
| 单段最长 | ≥200 字 | 80-150 字 | **120-180 字** |
| 配图密度 | 1.5-3/千字(15+ 张) | 每 1000 字 1 张 | **1.5-2/千字** |

**选择逻辑**:
- 追求「短期爆款 / 流量」→ 数据派(长文 + 多图)
- 追求「长期涨粉 / 完读 / 忠诚度」→ 学术派(短文 + 短段)
- 我们目标是涨粉 + 变现 → **偏向学术派**,但保留数据派部分优势

### 微信公众号 HTML/CSS 协议硬规则(Phase 3-2)

#### ✅ 允许的 HTML
`<p>` / `<section>` / `<h1-6>` / `<strong>` / `<em>` / `<br>` / `<blockquote>` / `<ul>` / `<ol>` / `<li>` / `<img>` / `<code>` / `<pre>`

#### ❌ 禁用
- `<iframe>` / `<script>` / `<style>` / `<form>` / `<video>`(全剥除)
- `<table>`(手机溢出,改用图片)
- `id` / `class` 属性(全删,只能 inline style)

#### 🚨 5 个核心坑

1. **position 属性** 全删 → 用 flex 替代
2. **`<style>` 块 + 外部 CSS** 全删 → **所有样式必须 inline**
3. **百分比单位**(transform/margin)失效 → 改 px / vw
4. ⭐⭐⭐ **段落默认 24pt 段后距(2022 后)**
   - 不覆盖会叠加你写的 margin → 视觉过空
   - **必须显式 `padding:0; margin:0 0 20px 0;`** 才能控制
5. **暗黑模式自动反色** → 浅灰文字不可见,要够深
   - color #333 OK,color #999 危险

#### 🚨 微信官方硬限制

```

                ≤ 1 MB
```
