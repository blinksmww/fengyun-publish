## Phase 2 · Musk 视角（物理 + Idiot Index）≤ 400字

**¥89 的 Idiot Index 是假的。**

计算3年 TCO 才是真物理学。火山引擎¥89/年看起来是最低 Idiot Index，但这个数字只是"第一年"的入场费，不是长期成本。官方从未承诺续费同价，有2023年已执行调价公告，同厂商 Coding Plan 涨价实证高达7.5倍。最坏情况：第二年续费¥1800+，3年 TCO 突破¥3700。这不是低 Idiot Index，这是隐藏的高杠杆赌注。

阿里云3年 TCO 是**¥597——确定数字**，不是概率分布。腾讯云3年乐观估计¥564，但"同价续费限1次"的模糊性让它实际值落在¥564到¥1020之间。

带宽维度：腾讯云5M固定、火山3M固定，阿里200M峰值。对月流量<5GB的项目，带宽差异**完全不影响物理运行**，不该是决策因素。

CPU维度：阿里云多核UnixBench 1747 vs 腾讯云1307，差34%。但fengyun-publish是I/O bound，CPU等待API返回，这34%差异对实际运行时间**影响接近零**。不应为此付溢价，也不应忽视这个性能数据——它证明阿里云在真正需要CPU时是备用储量。

Anthropic API代理¥20-50/月是三家公平的隐性成本，对厂商选择没有差异化影响，但**必须计入月成本**，很多人漏算这项。

**Musk verdict：选¥199阿里云。不是因为最便宜，是因为3年¥597是可以写进财务预算的确定数字。¥89是物理陷阱。**

## Phase 3 · Jobs 视角（产品稳定 + 长期TCO）≤ 400字

**续费同价才是真产品承诺，没有承诺就是没有产品。**

火山引擎的核心问题不是第一年贵不贵，是它卖给你的是一个**没有续费承诺的产品**。你买的是一年期服务，下一年的价格完全取决于他们的商业决定。这不是一个你可以在上面跑生产自动化系统的东西。fengyun-publish是每天1篇的24/7 cron——续费期到了发现涨价10倍，系统停摆，这一天文章没出，损失的是IP积累，不只是几百块钱。

腾讯云的"微信生态优势"在Phase 15调研里已经被核实：内网特权**只在CloudRun/CloudBase产品上**，轻量VPS走公网，与阿里云完全一样。Phase 14推荐腾讯云的核心理由"无缝对接微信"——这条在普通VPS场景不成立。腾讯云的真实优势只剩发票友好度一项。

发票维度：腾讯云"机构通用发票"是唯一对学校专门设计的格式，这是真实的差异化优势。但阿里云的普票在部分高校也有成功案例，且阿里在高校财务系统里的认知度同样高。两家都能报，只是腾讯云流程更顺。

文档与社区：这是被低估的隐性价值。部署遇到坑，能在5分钟内搜到答案vs需要30分钟，这个差异在大学生独立部署场景里是真实的时间成本。阿里云开发者社区国内最活跃，这不是广告，是可验证的事实。

**Jobs verdict：选阿里云。续费承诺是产品的一部分，没有承诺不是产品。文档和社区是生产系统的保险，不应该省这笔。**

### V3 · 3 个具体风险点 + Mitigations

**风险1：阿里云2027-03后续费涨价**

具体情景：2027-03-31同价承诺到期，第4年续费价格可能回归正式价¥822/年（约为活动价4倍）。

Mitigation：
- 设置日历提醒：**2027-01-15**（提前6周），届时主动登录控制台查看续费报价
- 若阿里云不续活动，备选迁移到腾讯云轻量（届时重新比价）；Docker容器化部署保证迁移成本 < 2小时
- 实例购买时选择**包年**，不选多年付——保持灵活性

**风险2：Anthropic API国内直连不稳导致cron失败**

具体情景：每天6:00 AM cron触发，Claude Code 9步流程在高峰期（晚间）遇到流式响应中断，当日文章未发。

Mitigation：
- **部署时必须配置**：在`.env`中设置`HTTP_PROXY=http://代理地址:端口`，推荐使用国内API中转服务（月¥20-50，已纳入TCO）
- cron时间设为**凌晨2-4点**（国内互联网低峰）而非白天或晚间高峰
- 在`daily_ship.sh`中加入重试逻辑：失败后30分钟自动重试，最多3次；仍失败则发企微告警
- 备选方案：阿里云百炼提供Anthropic兼容API（国内节点），可设为fallback

**风险3：学校报销发票格式不符**

具体情景：购买后发现学校财务系统要求单位抬头发票，但个人账户只能开个人名义普票，无法报销，¥199自费。

Mitigation：
- **购买前先问**：联系学校财务老师，确认「增值税电子普通发票（个人名义，阿里云）」是否接受——这一步5分钟，可避免后续纠纷
- 若财务说不接受：考虑将账号改为企业认证（填学校信息），或接受¥199自费（金额小，不值得折腾）
- 腾讯云的「机构通用发票」格式对学校更友好——若阿里报销卡住，可以临时切换腾讯云，额外代价只是迁移2小时工时

### V4 · 7天内具体操作步骤

**Day 0（今天，5月22日）**

1. 打开 aliyun.com/product/swas，找到「轻量应用服务器 2C4G 199元/年」活动套餐
2. 加入购物车，进入结算页
3. 在结算页尝试叠加300元学生券（控制台「我的权益」里找）
4. **截图结算页**，确认：能叠券 → 自付¥0；不能叠 → 全额¥199报销
5. 付款，选机房**华东（杭州）**，选 Ubuntu 22.04 LTS

**Day 1（5月23日）**

1. 实例启动后，记录公网IP
2. SSH进入实例，安装Docker + Docker Compose
3. 配置阿里云ACR镜像加速器（`/etc/docker/daemon.json`），加速拉取镜像
4. 拉取 we-mp-rss + RSSHub 镜像，验证容器能正常启动
5. 确定代理/API中转方案（搜索「国内Anthropic API中转」，月¥20-50，找一个可用的）

**Day 2–3（5月24–25日）**

1. 部署 TrendRadar + fengyun-publish 全套工具链
2. 在`.env`中配置：`ANTHROPIC_API_KEY`、`HTTP_PROXY`（代理地址）、`VOLCENGINE_IMAGE_KEY`、微信公众号 AppID/Secret
3. 微信公众号后台：在「IP白名单」添加VPS公网IP（aliyun.com/product/swas的IP地址）
4. 手动跑一次 `python3 generate_cover_by_template.py` 验证豆包封面生成
5. 手动跑一次 `bash daily_ship.sh` 验证全流程

**Day 4–7（5月26–29日）**

1. 设置 cron：`0 2 * * * bash /path/to/daily_ship.sh >> /var/log/fengyun.log 2>&1`（凌晨2点，低峰）
2. 观察连续3天的cron日志，确认Claude API调用成功率
3. 若有失败，在`daily_ship.sh`中加重试逻辑（wait 30m && retry × 3）
4. 配置快照策略：每天自动快照1次，保留7天（阿里云控制台 → 快照策略）
5. 设置日历提醒：2027-01-15「检查阿里云续费价格」

### V5 · 一票否决项（紧急切换条件）

以下任一条件触发，立即评估切换：

| 条件 | 阈值 | 动作 |
|---|---|---|
| 阿里云续费涨价 | 续费报价 > ¥400/年（当前¥199的2倍）| 评估迁移到腾讯云轻量（届时重新比价）|

| 代理服务不可用 | 连续3天cron失败率 > 50% | 切换代理服务；或临时改用阿里云百炼兼容API |
| 实例性能不足 | 连续1周cron运行时间 > 90分钟（原设计≤60分钟）| 升级到4C8G套餐（届时价格约¥450，在可报销范围内）|
