## 5. 给 multi-dim SOP v2 的实战建议

### 哪些 hotness feature 该进 SOP v2

- **`topic_hotness_90d`** (ρ=+0.499, 4/4 同号,正向): 长期 baseline,作为 SOP 二级权重
- **`topic_hotness_30d`** (ρ=+0.439, 4/4 同号,正向): **选题/Hook 层加分项**——主题 30d 热度高 → +10~+15;主题 30d 热度 = 0 (冷启) → -5
- **`topic_volume_30d`** (ρ=+0.095, 4/4 同号,正向): 热度量;高 volume 表示主题正在被多次讨论 → 选题层加分
- **`topic_account_diversity_30d`** (ρ=+0.080, 4/4 同号,正向): 多账号共写 → 大势信号 → 选题层加分
- **`topic_freshness`** (ρ=+0.075, 4/4 同号,正向): 新主题 vs 老主题

### 替换静态 SOP 规则的方案

- **替换 "标题含 Skills → +15" 这种静态规则**:用 `topic_hotness_30d × scale` 自动生成系数
- 公式建议: `topic_score = clip(topic_hotness_30d × 25 + topic_trend_30v90 × 30, -10, +20)`
  - topic_hotness_30d=0.4(主题正在爆): +10 基线
  - topic_trend_30v90=+0.2(主题升温): 再 +6
  - topic_hotness_30d=0(冷主题): -10
- **冷启动主题保护**: topic_volume_30d < 3 时,降权(置信度不足),用 90d 替代
- **大势识别**: topic_account_diversity_30d ≥ 3(4 个账号有 3 个在写) → 锁定为风口选题,planner 强烈推荐
