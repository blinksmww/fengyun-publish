## 调研锁定的 3 个关键事实

### 事实 1 · 99 计划商品不能叠加任何优惠券

阿里云官方规则:
> "99 计划特价实例不可与满减券、补贴券叠加"
> "本优惠券不可与产品折扣叠加使用"

**实测验证**: 用户在购买页操作时应付费用恒为 ¥99.00,无券抵扣项。

### 事实 2 · 学生 300 元券适用范围

**明确不适用**:
- 99 计划 / 38 元秒杀特价服务器
- 域名 / 虚拟主机 / 云市场产品 / 专有云

**适用**:
- 阿里云**部分公共云产品**(常规价 ECS、OSS、RDS 等)
- 「在学生用券专区内选择其他符合规则的 ECS 配置进行购买和抵扣」

**核心**: 不能用券抵 99 计划,但可用于其他正常价位产品。

### 事实 3 · 学生专享价 ¥79/年 e 实例(同款配置,更便宜)

搜索结果原话:
> "2026 年阿里云针对高校在校学生推出专属超低优惠,ECS 经济型 e 实例 2核2G 学生专享价 79 元/年,续费同价"

**配置完全相同**: 同样 ecs.e-c1m1.large 2vCPU 2GiB 经济型 e,只是入口在「学生用券专区」而不是「99 计划公开页」。

## Musk × Jobs 辩论

### Musk · 物理 + Idiot Index

- 99 计划 ¥99 vs 学生专享 ¥79 **同配置同实例**,差额 ¥20/年,三年续费同价 ¥60。**¥79 完胜**。
- 真正的 Idiot Index 考量是 ── **300 元券不要勉强用**:
  - 抵 ¥1009 高配 → 实付 ¥709 → **比 ¥99 贵 ¥610**(反向亏钱)
  - 抵 ¥459 轻量 → 实付 ¥159 → **比 ¥99 还贵 ¥60**
- 券留着用在 OSS / RDS / 域名 / DNS 等**未来一定会用**的服务上。

### Jobs · 产品稳定性 + 长期 TCO

- **续费同价是这次决策的灵魂**, 1 年噱头不算账,3-5 年 TCO 才是真东西。
- 两个续费同价方案的稳定性差异:
  | 方案 | 续费同价 | 稳定性 |
  |---|---|---|
  | 99 计划 ¥99/年 | 锁至 2029(3 年) | ✅ 不依赖身份,商业规则保证 |
  | 学生专享 ¥79/年 | 续费同价 | ⚠️ 是否依赖学生身份续期?未明示 |
- 关键风险点: **学生毕业后 ¥79 续费同价是否仍有效**?搜索未明确回答。如果毕业那天阿里云说「按 ¥1009/年续费」,这 ¥20 便宜就是炸弹。
- 但 ¥99 vs ¥79 一年 ¥20 真不是决策大头,**真正该想的是 2C2G 够不够**。fengyun-publish cron 峰值 2-2.5GB,2C2G + 4G swap 是**边界够用**。

## 300 元学生券的正确归宿(1 年内用掉)

**不要为了用券硬抵 ECS**,留着配套用在:

| 用途 | 月/年价 | 价值 |
|---|---|---|
| 域名 fengyun-cloud.com 等 | ¥55-80/年 | 公众号 IP 配套站点 |
| OSS 对象存储(数据库远程备份) | ¥20-50/月 | we-mp-rss / TrendRadar DB 备份,防 VPS 挂掉数据丢 |
| 云解析 DNS 个人版 | ¥19.9/年 | 配域名用 |
| RDS MySQL 99 计划版 | ¥99/年 | 后期 critic 数据持久化升级用 |
| 弹性公网 IP | 按量 | 备用 IP |

**300 元 ÷ 12 个月 ≈ 每月 25 元额度**,完全用得掉。

## 我犯的错(承认 + 修正)

之前推荐时凭印象说"用 300 元券抵 ¥99 = 0 元实付", **没核实 99 计划是否允许用券**, 违反了用户给的"事实驱动 > 印象"原则。

用户在购买页亲眼看到应付价不变后, 立刻指出, 救了一次假承诺。

**机制层修正**:
- 这类涉及**优惠券规则**的事实, 必须先 WebSearch / WebFetch 验证再推荐
- 不要假设"看起来合理"的叠加规则一定成立, 阿里云这类平台规则多变
