## 开头模板共性

### 钩子模式分类（共 4 种）

1. **时间锚 + 外部事件**（篇 A、D、F）：「昨天/前几天/X上/某访谈」，把读者定位在一个刚刚发生的现实事件里，钩子权威感来自外部消息源。
2. **时间锚 + 强主观体验**（篇 B）：「昨天，我突然……」，情绪词（顿悟/少有）密度高，钩子权威感来自个人经历的稀缺性。
3. **个人数据揭示**（篇 C）：直接甩自己的真实数据（131亿token/606个会话），没有解释也没有铺垫，让数字自己说话。
4. **读者默认假设点破**（篇 E）：以「你」为主语，描述读者习以为常的现象，然后悄悄埋下疑问（但到底存在多久了？），强迫读者反思。

### 平均开头字数

第一个自然段到首个断点：约 18–62 字，中位数约 37 字。无一超过 70 字，多数不超过 60 字。

### 出现的元素

- 时间锚词（昨天/前几天）：6 篇中 4 篇出现
- 第一人称「我」：5 篇出现（篇 E 例外，以「你」开头）
- 具体数字：4 篇在开头 3 段内出现具体数字（分数/时长/星数/小时数）
- 外部权威引述（第三方机构/知名人物）：3 篇

### 不出现的元素

- **不出现**「大家好/今天我们聊聊/关注我/欢迎来到」等套话
- **不出现**「先说结论」模式（6 篇均无显式结论先行句式）
- **不出现**纯疑问句开头（没有一篇以问号起首段）
- **不出现**格言/名言引用作为第一段

## 结尾模板共性

### 收尾结构模板（4 种）

1. **行动指引型**（篇 A）：告诉读者下一步可以做什么，不带「如果觉得有用请点赞」。指令具体可执行（「亲自上手跑一遍」）。
2. **点题留白型**（篇 B、F）：一句话收拢全文核心命题，戛然而止。不解释，不追加。字数极短（10–16 字）。
3. **警句型**（篇 C）：用加粗强调的一句反常识警告作结。结论从作者个人经验里提炼，带「你review任务的边际成本不是0」这类逻辑密度高的句子。
4. **开放邀请型**（篇 D、E）：不下定论，表达个人态度后邀请读者一起思考。句式接近「我不知道……但我挺期待看到……」。

### H2 前后的视觉节奏

**H2 前**：没有显式空段分割符或图形分割线（篇 C 末尾参考来源前有一条 `<hr>` 渐变横线，篇 D 末尾有一条 `<hr>`，均在正文最末，不是章节间分割）。章节标题直接跟在上一段正文之后，靠 CSS `margin-top: 32px` 制造视觉间距，无额外空行或装饰性元素。

**H2 后第一段的开头模式**：
- 直接展开（最常见）：约占 70%，不过渡直接切入主题
- 承接上文再展开：约占 20%，用一句话收住上文然后转入新方向（「Agent View本质是个产品功能。」）
- 个人经历/私事引入：约占 10%（篇 D 的「先讲一条我最近一个月……调整的一条流程规则」）

## 过渡句库

以下为 6 篇正文中高频出现的承上启下句式（含出处篇名）：

| 过渡句原文 | 出处篇 | 功能 |
|------------|--------|------|
| 「所以今天想聊聊这事。」 | D | 从引子转入正文 |
| 「我提这个，主要是想说个现状，就是：」 | C | 铺垫结论前的预告 |
| 「写到这，收一下。」 | C | 从讲解转结尾 |
| 「下面就实测一下这事。」 | A | 从理论铺垫转实操 |
| 「回头看……感受会平实一些。」 | A | 从细节回归主题 |
| 「讲到这里，必须提一个……」 | E | 引入新人物/概念 |
| 「但有一件事必须做，做了和没做差别巨大：」 | B | 强调转折 |
| 「真正的问题在哪？」 | B | 反问引出核心论点 |
| 「读到 MiniMax 官方对 Mavis 的描述时我笑了一下。」 | D | 个人反应作转场钩 |
| 「把这四个放一起看，Mavis做的事就清晰了。」 | D | 列举后的收拢总结 |
| 「我装上之后第一反应是兴奋，第二反应是失望。这篇就讲这两件事，再讲……」 | F | 预告式路线图 |
| 「兴奋玩了一下午后，我决定用它做件正经事：」 | F | 从演示转到问题发现 |
| 「写到这里我想到了……」 | D | 个人思维切换的显式标注 |
| 「回到开头那个问题……」 | E | 首尾呼应式复归 |
| 「但越想越觉得，这才是它真正的礼物。」 | B | 反向加深核心论点 |

**高频句式规律**：
1. 「所以……」引出行动或结论（约每篇出现 2–4 次）
2. 「但……」转折，多用于引出反例或更深一层的观察（约每篇出现 3–6 次）
3. 「这件事……」指代前文事物并延伸（高密度，每篇数十次）
4. 「……其实……」：揭示反直觉信息（每篇约出现 2–3 次）
5. 「翻译成……就是」：把专业术语或抽象概念转化为日常语言（每篇约出现 1–3 次）

## 关键发现

### 花叔开头

- **不用**「大家好/今天来聊聊/欢迎大家」等开场白，0 篇出现
- **不用**「先说结论」句式，0 篇出现
- **不用**纯疑问句作第一段
- **不用**名言/格言作第一段
- **常用** 时间锚词（昨天/前几天）：4/6 篇
- **常用** 第一段≤60字：6/6 篇，极短入口
- **常用** 开头即甩具体场景/数据/事件，不铺背景
- **常用** 5 篇中有 1 篇（篇C）在正文前插入题眼（斜体 italic 导语），功能类似编辑标注而非正文

### 花叔结尾

- **不用** 转发/点赞/关注引导，0 篇出现
- **不用** 二维码、footer，0 篇出现
- **不用** 彩蛋式问答（「你怎么看？欢迎评论区聊聊」之类），0 篇出现
- **常用** 极短末段（10–60字）：6/6 篇末段均不超过 65字
- **常用** 留白或开放型收尾（不给答案，或者给一个有张力的半问句/个人表态）：4/6 篇
- **常用** 末段与标题/首段有回声关系（直接复用关键词或主题）：4/6 篇
- **偶尔** 行动指引型收尾（鼓励动手，不带求赞）：1/6 篇（篇 A）
- **偶尔** 警句型收尾（一句加粗格言式总结）：1/6 篇（篇 C）

### H2 标题规律

- 最常见 3 种句式：**概念陈述句**（含「是/不是」的判断句）、**数字编号+简洁名词短语**（一/二/三格式）、**口语化描述句**（含动词，像说话一样）
- 标题长度：5–25 字，没有超长标题
- 标题不使用「：如何……」「XXX的N种方法」等内容营销模板格式
- 篇 B 极端案例：所有章节标题仅用「一」「二」「三」「四」「五」，不加描述

### 过渡句最高频

- 「但……」转折句（每篇高密度，功能为引出反例或更深洞察）
- 「翻译成……就是……」（专业→日常的语义降维句）
- 「这件事……」（指代延伸句，保持话题连贯）
- 「写到这，收一下」（显式收束章节，篇 C）
