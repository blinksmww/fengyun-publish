## 关键发现（只列事实）

1. **正文字号 17px，6/6 篇一致**，均带 `!important`。
2. **正文行高 1.8，5/6 篇一致**（篇6为1.85，差值0.05）。
3. **正文颜色：篇1-5 使用 `#2b2b2b`（5篇一致）；篇6 使用 `#1a1a1a`（独立方案）。**
4. **主强调色：篇1-5 统一 `#C15F3C`（赤陶橙）；篇6 为 `#d32f2f`（鲜红），两套完全独立。**
5. **页面背景：篇1-5 为暖米色 `#faf9f7`；篇6 为纯白 `#fff`。**
6. **正文段间距 `margin: 20px 0 !important`，5/6 篇一致（篇6为18px）。**
7. **图片统一：`max-height: 500px !important`、`border-radius: 10px`、`margin: 24px auto`、`display: block`，6/6 篇完全一致。**
8. **`<strong>` 行内高亮块格式：`padding: 2px 6px; border-radius: 3px`，6/6 一致；颜色跟随各篇主强调色。**
9. **列表间距：`ul/ol margin: 20px 0; padding-left: 28px; li margin: 10px 0`，6/6 一致。**
10. **字间距：正文 `-0.005em`，5/6 篇（篇1-5）使用；篇6正文无此属性。**
11. **图片阴影：篇1-5 用 `rgba(193,95,60,0.1)` 暖橙调阴影；篇6 用 `rgba(211,47,47,0.12)` 暖红调阴影，均与各自主色呼应。**
12. **最外层容器 `max-width: 700px; margin: 0 auto`，6/6 一致。**

## 不确定的地方

1. **篇6（微信读书文）是否为同一作者设计**：篇6在配色（背景、主色、强调色）、字号（h2 20px vs 26px）、行高（正文1.85 vs 1.8）、padding（更紧凑）上全面偏离其他5篇，判断为使用了另一套排版模板，是否出自同一设计者/pipeline无法从HTML本身确认。

2. **`<h2>` 渐变文字在微信客户端的实际呈现**：使用了 `-webkit-background-clip: text` 技术，该属性在部分安卓微信 WebView 版本下可能降级为纯色显示。本报告仅统计 HTML 中的声明值，未做实机渲染验证。

3. **外部 CSS 文件内容未纳入统计**：每篇 HTML 引用约 18 个外部 `.css` 文件（assets 目录），这些文件中可能包含对页面结构元素（如 `h1.rich_media_title`、`body` 基础字号等）的额外样式规则。本报告仅统计 `#js_content` 内的 inline style，外部 CSS 的影响未知。

4. **`<p>` 内嵌 `<span leaf="">` 的字号继承**：部分 `<p>` 元素内有多个 `<span leaf="">` 子元素，这些 span 无独立 style 声明，字号依赖父级 `<p>` 的 17px 继承。但个别位置（如篇3的导语行）有 `style="font-size: 15px"` 独立设置的 span，此类情况出现次数较少未完整统计。

5. **table 在5篇中的分布**：表格（`<table>`）仅在篇1（Qwen3.7文）、篇3（Agent View文）、篇4（Mavis文）、篇6（微信读书文）出现，篇2（AI失败文）和篇5（公司文）无表格。表格样式统计中「4/6」指这4篇。
