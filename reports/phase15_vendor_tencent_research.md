## 3 · 配置规格细节

### CPU
- 型号：Intel® Xeon® Cascade Lake（2.5GHz 基础频率，3.1GHz Turbo Boost）或 Cooper Lake
- **注意**：轻量应用服务器不支持指定 CPU 型号，系统自动分配；通常优先分配较新一代 CPU
- 与阿里云对比：阿里云 2C4G 轻量使用 Intel Xeon Sapphire Rapids（更新一代），性能略优（调研 3 实测：阿里云 CPU 密集任务约快 6%，MySQL TPS 约高 7.1%）

### SSD 存储
- 类型：全 NVMe SSD 云盘，三副本分布式
- IOPS：**官方未公布轻量应用服务器的具体 IOPS 上限**（仅说明「低延迟、高随机 IOPS」）
- CVM 标准型 IOPS 参考：高性能云盘约 2400 IOPS，SSD 云盘约 16000+ IOPS

### 带宽
- 2C4G 标配：**5Mbps 固定带宽**（非突发）
- 买1送3版：6Mbps 固定
- **项目评估**：月流量 < 5GB，5Mbps 带宽绝对够用（即便持续满速写入，100 小时才 5GB）
- 对比阿里云 2C4G 轻量：200Mbps 峰值 + 不限流量（但对本项目无实际价值）

### 流量规则
- 超出 500GB/月后：超量费 0.5-1.0 元/GB（境内）
- fengyun-publish 月流量 < 5GB，**永远不会触发超量费**

## 5 · 发票 + 报销

### 发票类型
- **数电发票**（全电子）：PDF / OFD / XML 三种格式，提交后约 30 分钟到邮箱
- 支持类型：
  - 增值税普通发票（个人认证账户可开）
  - 增值税专用发票（需企业认证）
  - 机构通用发票（适用于学校、事业单位、非盈利组织）

### 学校报销兼容性
- **机构通用发票**专门适配学校/事业单位报销场景，不能抵扣增值税进项但完全符合学校报销要求
- **全电子发票**（数电发票）是当前国家标准，主流学校报销系统均已支持
- 个人账户（实名认证为个人）可以开增值税普通发票，抬头填写学校名称
- **对比火山引擎**：腾讯云发票体系完善度明显优于火山引擎（火山引擎对学校报销兼容性存疑）

### 报销流程
1. 腾讯云控制台 → 费用中心 → 发票管理 → 申请开票
2. 选择「按消费明细」或「按月账单」
3. 约 30 分钟后电子发票到达注册邮箱
4. 直接上传学校报销系统

## 6 · 网络质量（项目关键维度）

### 核心问题：腾讯云调用微信公众号 API 是否有延迟优势？

#### 普通 VPS 调用路径（阿里云 / 腾讯云 CVM）
- 路径：VPS 公网 IP → 互联网 → api.weixin.qq.com（腾讯数据中心）
- 腾讯云与微信 API 服务器物理上同属腾讯网络，**但轻量应用服务器走的是公网出口，并非内网直连**
- 微信 API 官方检测接口（`callback/check`）回显延迟约 20-30ms（国内公网环境）

#### 微信云托管（CloudRun）调用路径（真正的内网特权）
- 路径：微信云托管容器 → **内网代理** → api.weixin.qq.com（无需 IP 白名单）
- 「开放接口服务」：以旁加载形式与容器共存，**自动中转 `api.weixin.qq.com` 请求，自动处理 token 和鉴权**
- access_token 每 10 分钟推送一次，挂载在 `/.tencentcloudbase/wx/cloudbase_access_token` 路径
- 容器内可以**不携带 access_token 参数**直接调用微信 API——由旁加载服务自动注入

#### 结论：「内网优势」是**有条件的实质优势，而非纯营销话术**
- **如果用腾讯云轻量应用服务器（普通 VPS）**：与阿里云相比，调用微信 API 的延迟**没有实质性优势**，都走公网
- **如果用微信云托管（CloudRun）或 CloudBase**：有真实内网特权——无 IP 白名单限制 + access_token 自动托管 + 延迟更低
- **fengyun-publish 的实际影响**：当前架构用 `post_fengyun_publish.py` 走公网 REST API，腾讯云普通 VPS 不提供内网优势。若迁移到微信云托管则有实质优势，但架构改动较大

### 对豆包 Seedream API（火山引擎）的延迟
- 调用地址：`ark.cn-beijing.volces.com`（字节跳动北京数据中心）
- 腾讯云与火山引擎无特殊互联关系，延迟与阿里云相当（国内公网 30-50ms）

### 对 Anthropic API 的国际出口
- Anthropic API：`api.anthropic.com`（美国）
- 国内三大云厂商（阿里/腾讯/华为）的国际出口延迟差异不大，均通过国内 CDN/运营商出口
- 预计延迟：200-400ms（正常 API 调用不敏感）

## 7 · 微信生态原生集成（核心卖点验证）

### CloudBase（腾讯云开发）是否原生支持公众号 API

**是的，有实质性集成，但仅限于特定产品**：

| 产品 | 内网调用 | access_token 自动托管 | IP 白名单免除 |
|---|---|---|---|
| **微信云托管（CloudRun）** | ✅ 真实内网 | ✅ 自动推送挂载 | ✅ 无需配置 |
| **CloudBase FaaS** | ✅ 真实内网 | ✅ 自动获取 | ✅ 无需配置 |
| **腾讯云轻量 VPS** | ❌ 走公网 | ❌ 需自行维护 | ❌ 需配置 IP 白名单 |
| **腾讯云 CVM** | ❌ 走公网 | ❌ 需自行维护 | ❌ 需配置 IP 白名单 |

### 企微 webhook 是否享 LAN 内网延迟
- 企微 webhook（`qyapi.weixin.qq.com`）同样是腾讯系服务
- **普通 VPS 调用**：走公网，约 20-30ms，无内网特权
- **CloudRun 内调用**：理论上可以更低延迟，但官方文档未明确说明企微 webhook 也有旁加载内网代理

### 与 fengyun-publish Step 8（推草稿）的原生集成方案

**有集成方案，但需要架构迁移**：

**方案 A（当前架构，轻量 VPS）**：
```
post_fengyun_publish.py → 公网 → api.weixin.qq.com/cgi-bin/draft/add
需要：IP 白名单 + 自行维护 access_token
```

**方案 B（迁移到微信云托管）**：
```
容器内代码 → 内网旁加载 → api.weixin.qq.com/cgi-bin/draft/add
不需要：IP 白名单 / access_token 管理
```

**方案 B 的代价**：
- 需要将整个 fengyun-publish cron 架构从 VPS 迁移到微信云托管容器
- 微信云托管按请求计费，长时间 Claude Code 进程可能不适合容器化
- **Phase 11 已验证**：轻量 VPS + IP 白名单方案完全可行，迁移云托管并非必须

## 10 · 针对 fengyun-publish 的综合评估

### 3 优

1. **发票报销流程最友好**：数电发票 30 分钟到邮箱，机构通用发票完全适配学校报销系统，优于火山引擎。
2. **微信生态集成存在实质能力**：若未来迁移到微信云托管，可获得 access_token 自动托管 + 无 IP 白名单限制的真实内网优势，是三家厂商中唯一有此能力的。
3. **「买1年送3月」活动性价比高**：¥188/年等效 15 个月，折合 ¥150/年，比阿里云 ¥199 省 ¥50；新老用户均可参与。

### 3 劣

1. **普通 VPS 不享有微信 API 内网优势**：轻量应用服务器（fengyun-publish 实际部署目标）与阿里云一样走公网，「腾讯系亲生」对普通 VPS 用户是营销话术，实质内网特权只在云托管/CloudBase 产品上体现。
2. **性能略低于阿里云**：CPU 为 Cascade Lake（略旧），实测比阿里云慢 6-7%；IOPS 未公开，透明度低于阿里云。
3. **续费同价政策有「限 1 次」模糊性**：「同价续费限 1 次」在部分套餐中存在，第二次续费机制不够透明，长期成本有不确定性。

## 关键裁定：微信生态原生集成是实质优势还是营销话术？

### 裁定：**有条件的实质优势，但对 fengyun-publish 当前架构不适用**

| 场景 | 实质性 | 说明 |
|---|---|---|
| 腾讯云轻量 VPS + `post_fengyun_publish.py` | ❌ 无优势 | 走公网，与阿里云无差异，仍需 IP 白名单 |
| 微信云托管（CloudRun）容器化部署 | ✅ 实质优势 | 内网调用，无需 IP 白名单，access_token 自动托管 |
| CloudBase FaaS 函数 | ✅ 实质优势 | 同上 |

**结论**：Phase 14 报告中「腾讯云原话『无缝对接微信小程序』」的描述适用于云托管产品，而 fengyun-publish 的部署目标是**普通 VPS（轻量应用服务器）**。在当前架构下，腾讯云轻量与阿里云轻量对微信 API 的调用体验没有实质差异。若未来考虑迁移到云托管，腾讯云的生态优势才会真正体现。
