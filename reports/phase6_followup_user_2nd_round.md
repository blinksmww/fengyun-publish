## Musk × Jobs 对话(4 轮)

**Musk(第 1 轮 — 先看物理事实，再算约束)**

我先不表态，先算事实。

第一个事实：Phase 6 已经有一张活证据——`baoyu_v2_T1_agent.png`，主标题写的是"Anthropic 一周 7 件事"，Anthropic 是英文，13 个字母，完整出现在图右半大字区，拼写**完全正确**，没有任何漂移。这张图正是在当前的 prompt 体系下生成的。

第二个事实：同一批生成里，`baoyu_v2_T2_research.png` 里"Antropic"少了一个 h——同一套 prompt，同一个模型，一张对一张错。

结论：英文准确率不是 0% 也不是 100%，是**概率问题**，约 50-80%，受 seed + 渲染随机性影响。纯中文化能把这个概率从 80% 推到 98%，但风云拒绝，说明他认为"品牌名在主标"的信息价值 > 80%→98% 的准确率收益。

这是一个合理的用户需求。那我们的问题就变成：**如何在保留英文主标的前提下，把英文准确率从 ~70% 拉到 95%+？**

物理约束清单：
- Seedream 5.0 的文字渲染不是确定性输出，不存在"100% 准确"的技术保证
- 英文单词越长，漂移概率越高（"GPT"→几乎不错，"Anthropic"→13 字母，Idiot Index 最高）
- 重复次数在 prompt 里增加 → 模型对文字的注意力权重提高 → 准确率可以改善，但有边际递减

工程路径：**prompt 加强 + OCR retry 组合**，这是唯一可以在不破坏风格、不做技术降级的前提下把准确率推到 95%+ 的方案。

**Jobs(第 2 轮 — 挑战：craft 维度的英文精度问题)**

风云说得对，英文品牌名是信息，不是装饰。我来说 craft 维度的意见。

我刚看了 T1_agent.png。"Anthropic" 在图里以大字形式完整正确呈现——这是可以做到的。问题是它做到的概率只有约 50%。这 50% 成功率不是品质，是运气。靠运气交付的产品没有灵魂。

有人会说"PIL 合成文字层"——我明确反对这个方案。如果标题文字是后期贴上去的，整张图的 craft 完整性就破掉了：手绘背景 + 合成文字，读者看缩略图感觉对，放大一看文字和图的质感不统一。这不是 mediocre，这是设计上的谎言。

正确的方向：**在 Seedream 的概率渲染里用工程手段筛选出正确的那张**，而不是技术降级绕过问题。

具体说：
1. **prompt 里英文重复 5 次**（现在是 3 次）——把标题 MUST 出现在 prompt 的 5 个不同位置，包括：场景描述、右侧标题行、MUST READ EXACTLY 行、negative prompt 的镜像（"DO NOT write Antro-pic or Antrompic"）、最后的质量强化行
2. **多 seed 并发生成 3 张，OCR 校验取最佳**——不等 retry，直接一次调 3 个不同 seed，第一张 OCR 通过就停，不通过就看第 2、第 3 张
3. **拼写错误负向 prompt**——每次生成前把常见 typo 列在 negative prompt 里：`NEGATIVE: Antrompic, Antropic, Anthrobic, Anthrop1c`

这不是完美，但这是 craft 诚实的方案。

Musk 在参考图上的分析是对的，但我要补一个 craft 约束：**追加的参考图必须是真正高质量的**，数量是为了充足先验，不是为了堆砌。

## 共识方案

### Part A · 英文准确性保证(替代纯中文化)

风云否决纯中文化后，采用以下三层工程方案：

**选定方案 1：prompt 英文重复强化（从 3 次 → 5 次）+ typo negative prompt**

在每个模板 prompt 里，标题的英文品牌名至少出现在 5 个位置：
1. 场景描述中自然出现一次
2. RIGHT 区标题行精确写出（带引号）
3. `MUST read EXACTLY: "Anthropic 一周7件事"` 行
4. Negative prompt：`DO NOT misspell: not "Antrompic", not "Antropic", not "Anthrobic"`
5. 最末质量强化行：`Render ALL text characters precisely and correctly`

**选定方案 2：多 seed 并发 3 张 + OCR 校验取最佳**

不等 retry（串行），直接并发生成 3 张不同 seed，OCR 校验找第一张标题拼写全部正确的返回。预期：3 次并发中至少 1 张正确的概率 = 1 - (0.3)³ = 97.3%（以单张失败率 30% 保守估算）。

代价：API 调用成本 × 3（豆包 Seedream 成本约 ¥0.1-0.3/张，3 张约 ¥0.3-0.9，可接受）。

**实施代码改动：**

| # | 改动 | 文件 | 工时 |
|---|---|---|---|
| 1 | 每个模板 prompt 末尾加 negative typo 行 | `tools/generate_cover_by_template.py`，各模板 prompt 字符串 | 10 min |
| 2 | 英文标题出现从 3 处增到 5 处（补场景描述首句 + 末尾质量强化） | 同上，各模板 prompt | 10 min |
| 3 | `generate_image()` 改为并发生成 3 张（seed+0/+1/+2），OCR 检测英文关键词 | `generate_cover_by_template.py` 235-278 行 | 30 min |
| 4 | OCR 模块：优先用 pytesseract（已知系统装了 Tesseract），检测英文单词拼写是否与标题一致 | 新增 `tools/cover_ocr_verify.py` | 20 min |
| **合计** | | | **70 min** |

**PIL 文字合成层：明确放弃。** 手绘背景 + 合成文字质感不统一，破坏 craft 完整性，不采用。

### Part B · T1 架构图风格扩展

**结论：T1 保持专化，其他 4 个模板不改动。**

理由：
- T1 的"节点辐射/同心圆"构图是 Agent 系统类文章的专属视觉语言，天然匹配风云"研究 Agent 的云"的 IP 定位
- T5_method 的构图是"思考人物 + 多卡片"，如果改成同心圆会和 T1 撞型，5 个模板的差异化价值消失
- T1 专化 = 当文章主题是"Agent 架构/系统/能力拆解"时自动路由到 T1，得到节点图构图；其他类型文章用对应模板

**不做的事：** 不把同心圆结构套入 T2/T3/T4/T5。

**可选的微调（非 P0）：** T1 的 classify() 关键词可以扩充，确保"架构/系统/节点/能力/harness/组件/层"等词全部命中 T1，而不误路由到 T5。这是 5 分钟的关键词表更新。
