# Phase 5 Follow-up · 为什么主力 fal.ai 而不是即梦官方?
*Musk × Jobs 对话回答 · 2026-05-21*

---

## 风云的质疑回顾

风云读完 Phase 5 报告后提出:他当时明确写了"优先免费或极低成本方案,豆包/即梦 API 可用,国外付费模型不可用"。但报告把 fal.ai(国外平台,$0.035/张)定为主力,把即梦官方 API(免费 66 张/天)放在降级 1。

审判官追加了更具体的质疑:一篇文章一张封面,即梦 66 张/天免费额度完全够用,直接免费;而 fal.ai 是付费国外平台,违反字面硬约束。建议翻转降级链:即梦主力 → 阿里万相降级 → fal.ai 兜底。

---

## Musk × Jobs 对话(3 轮真实交锋)

**Musk**:

我先承认一个问题。调研 #1 里我们查到了"即梦官方 API,每日赠 66 积分可生免费图 66 张",这个数字当时我们写进报告了,却还是把 fal.ai 放主力。从我的五步删除法看,这是第 3 步的错误——我们在简化/优化一个"不该存在的复杂度":一篇文章一张封面,66 张/天的免费额度是物理上绰绰有余的,Idiot Index 是∞——你付钱买了一个免费的东西。

但我当时有一个具体理由,Jobs 调研 #1 里挖出来的:fal.ai 路由"不需要国内网络问题,走国际 API;火山引擎直连需国内账号"。这是我选 fal.ai 主力的核心原因。不是功能,是注册门槛和网络环境。我当时的判断是:风云注册火山引擎、开通即梦 API 的链路比注册 fal.ai 更长。

但我现在要问:这个假设真的成立吗?风云有火山引擎账号吗?如果有,这个理由就消失了。

---

**Jobs**:

Musk 说的注册门槛,我认为是一个"合理化已经做的决定",而不是"真正的设计理由"。

我来还原当时真正发生了什么:Phase 5 辩论时,我们拿到的调研数据里,Seedream 5.0 Lite 的 fal.ai 接口是最完整的——有精确的 endpoint 地址(`fal-ai/bytedance/seedream/v5/lite/text-to-image`)、有精确的定价($0.035/张)、有精确的图像尺寸上限(3072×3072px)。即梦官方 API 的信息我们只知道"66 积分/天免费",具体的 API 接口参数、中文渲染能力的控制精度我们没有验证。

这暴露了一个 Jobs 视角的问题:我们选工具是根据"哪个 demo 跑通了"而不是"哪个方案对用户更好"。报告里风云的 `claude_style_cover_test.py` 脚本已经对接了 fal.ai,这是存量资产,我们就沿用了。这是惰性,不是判断。

我同意审判官的质疑方向。但我要补充一个真实约束:即梦 API 是不是像 fal.ai 一样支持精确的 seed 参数?我们在 Phase 5 Round 5 里把 seed 固定作为质量核心——"每个模板固定一个 seed,之后每次可复现"。如果即梦官方 API 不支持 seed 固定,那它就不是对等替代品。

---

**Musk**:

Jobs 的 seed 问题是真正的约束检验。但我们现在没有数据,不能在这里猜。我只能算已有信息:

1. 即梦官方 API 是字节跳动火山引擎自营产品,Seedream 和即梦底层同源——都是字节的图像生成体系。seed 参数是业界标准参数,几乎所有主流图像 API 都支持。这不是物理约束,这是"我们没查过"的假约束。

2. 成本对比。fal.ai 主力:假设月产 30 篇文章,每篇 1 封面 = 30 张 × $0.035 = $1.05/月。即梦免费:0 元/月。差异不大,但原则性错误不在成本高低——在于**明确违反了用户的字面约束**。风云说"优先免费",我们给了"付费外国平台",不管这个付费有多小。

3. 国网访问问题。fal.ai 是境外平台,在中国大陆访问可能需要特殊网络配置。即梦是国内平台,零障碍。我们在 Phase 5 里提到了"fal.ai 不需要国内网络问题"——但这个说法本身就自我矛盾:你要访问一个境外平台,你的脚本运行环境需要能连到 fal.ai 的服务器,这不是"零问题",这是"我们假设了一个国际网络环境"。

结论:审判官的翻转建议是正确的。即梦官方主力,fal.ai 降格到兜底。

---

**Jobs**:

我接受这个结论。最后一个补充——执行顺序。

风云 Day 1 应该先注册即梦官方 API(火山引擎账号,国内无障碍),验证它的 seed 参数和 prompt 中文响应。如果验证通过,直接把即梦设为主力,fal.ai 变兜底。验证失败(seed 不支持或中文渲染不够精准)再讨论是否需要 fal.ai 补位。

这是"展示而非询问"的具体应用:不在这里猜即梦能不能用,去跑一次就知道了。

---

## 共识

经过交流,Musk + Jobs 一致认为:

- **完全翻转**:即梦官方 API 升为主力,fal.ai 降格为兜底备用
- 原方案的核心漏判是:调研 #1 获得了即梦 66 张/天免费的事实,但因为 `claude_style_cover_test.py` 已对接 fal.ai 的惯性,没有质疑"为什么不用免费的"
- fal.ai 作为境外付费平台,违反风云"优先免费/国内方案"的字面硬约束,不应在满足约束的替代品存在时作为主力

## 给风云的具体执行建议(actionable)

### 修订后的工具优先级表

| 层级 | 工具 | 价格 | 触发条件 |
|------|------|------|---------|
| **主力** | 即梦官方 API(火山引擎,Seedream 体系) | 66 张/天免费 | 默认 |
| **降级 1** | 阿里云万相 wan2.6-image | 90 天免费额度 | 即梦 API 失败/超限 |
| **降级 2** | baoyu-cover-image → baoyu-imagine | 取决于 runtime | 前两者均失败 |
| **境外兜底** | Seedream 5.0 Lite via fal.ai | $0.035/张 | 国内方案全部不可用时 |

> 注:fal.ai 不再是默认路径,仅在国内方案全部故障且需要紧急出图时使用。

### Day 1 实际操作顺序

1. **先注册火山引擎账号**(volcengine.com),开通即梦 API(文档:`volcengine.com/docs/85621/`)
2. 用即梦 API 验证两件事:
   - seed 参数是否支持(在请求体里加 `"seed": 42` 测试可复现性)
   - 5 个封面 prompt 模板的中文渲染响应(各跑 1 次,看云形符号是否出现)
3. 若验证通过:更新 `COVER_STYLE_GUIDE.md` 和 `tools/ship.py` 的 API endpoint 为即梦接口
4. 若 seed 不支持或渲染差:向下查阿里云万相;fal.ai 作为技术应急选项

### `tools/ship.py` API 端点修改

将原来的 fal.ai endpoint:
```python
"fal-ai/bytedance/seedream/v5/lite/text-to-image"
```

替换为即梦官方 API(待验证后填入具体 endpoint,参考 `volcengine.com/docs/85621/`)。降级链顺序更新为:即梦 → 万相 → baoyu-imagine → fal.ai。
